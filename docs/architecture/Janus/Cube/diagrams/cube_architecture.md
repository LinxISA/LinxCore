# CUBE 架构图（Janus v2.0）

本文档提供 Janus CUBE v2.0 的架构图和流水线图。

## 2. CUBE 数据流图

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         tmatmul / tmatmul.acc                            │
│                                                                          │
│  BCC → 命令解码 → Tile Cmd Buffer (4-deep) → FSM Fractal 拆分           │
│                        ↓                                                 │
│                    计算 uop 数量                                         │
│                    M_tiles × N_tiles × K_tiles                          │
│                        ↓                                                 │
│  ┌──────────────────────────────────────────────────────────────────┐  │
│  │                 ISQ (32 uop)                                      │  │
│  │                                                                   │  │
│  │  uop[i,j,k] 入队:                                                 │  │
│  │    - 计算 A_addr, B_addr                                          │  │
│  │    - 分配 ACC slice                                               │  │
│  │    - 建立 K 方向依赖                                              │  │
│  │                                                                   │  │
│  │  依赖检测:                                                         │  │
│  │    - K_idx > 0: deps = uop[i,j,k-1]                              │  │
│  │    - src_ready: 查询 L0A/L0B Cache                               │  │
│  │                                                                   │  │
│  │  调度策略:                                                         │  │
│  │    - K_idx==0: 高优先级                                           │  │
│  │    - src_ready && deps_ready: 可发射                             │  │
│  │    - 乱序选择（MN 方向）                                          │  │
│  └────────────────────────┬─────────────────────────────────────────┘  │
│                           ↓                                             │
│  ┌──────────────────────────────────────────────────────────────────┐  │
│  │               L0A/L0B Cache 查找                                  │  │
│  │                                                                   │  │
│  │  Hit:  读取数据 → 发送到 MAC                                      │  │
│  │  Miss: 发送 tileload 请求 → 等待填充 → 重试                       │  │
│  └────────────────────────┬─────────────────────────────────────────┘  │
│                           ↓                                             │
│  ┌──────────────────────────────────────────────────────────────────┐  │
│  │               MAC 阵列 (16×16)                                    │  │
│  │                                                                   │  │
│  │  A (512 B) ──→  ┌─────┐                                          │  │
│  │                 │ MAC │ ──→ C (1 KB, nz)                         │  │
│  │  B (512 B) ──→  └─────┘                                          │  │
│  │                                                                   │  │
│  │  延迟: ~7 cycles (脉动)                                           │  │
│  └────────────────────────┬─────────────────────────────────────────┘  │
│                           ↓                                             │
│  ┌──────────────────────────────────────────────────────────────────┐  │
│  │               BufferC 累加                                        │  │
│  │                                                                   │  │
│  │  if (K_idx % K_chunk == 0):                                      │  │
│  │      BufferC = MAC_output                                        │  │
│  │  else:                                                            │  │
│  │      BufferC += MAC_output                                       │  │
│  │                                                                   │  │
│  │  if ((K_idx+1) % K_chunk == 0 || last_K):                       │  │
│  │      ACC[slice] += BufferC  (RMW, 9 cy)                         │  │
│  └────────────────────────┬─────────────────────────────────────────┘  │
│                           ↓                                             │
│                      uop 完成                                            │
│                      广播 uop_id                                         │
│                      Wakeup 依赖的 uop                                   │
└─────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────┐
│                              acccvt                                      │
│                                                                          │
│  BCC → 命令解码 → Tile Cmd Buffer (4-deep) → acccvt 拆分                │
│                        ↓                                                 │
│                    查询 ACC 映射表                                       │
│                    slice_list = acc_mapping[chain][acc]                 │
│                        ↓                                                 │
│  ┌──────────────────────────────────────────────────────────────────┐  │
│  │          生成 acccvtuop (每个 slice 一个)                         │  │
│  │                                                                   │  │
│  │  acccvtuop[i]:                                                    │  │
│  │    - slice_id: slice_list[i]                                     │  │
│  │    - deps: matmul_uop[i]                                         │  │
│  │    - tile_dst: 目标 TileReg                                       │  │
│  │    - mode: 转换模式                                               │  │
│  └────────────────────────┬─────────────────────────────────────────┘  │
│                           ↓                                             │
│  ┌──────────────────────────────────────────────────────────────────┐  │
│  │               等待 matmul uop 完成                                │  │
│  │                                                                   │  │
│  │  matmul uop 完成 → 广播 uop_id                                    │  │
│  │  acccvtuop.deps_ready = 1                                        │  │
│  └────────────────────────┬─────────────────────────────────────────┘  │
│                           ↓                                             │
│  ┌──────────────────────────────────────────────────────────────────┐  │
│  │               FixPipe (11 级流水线)                               │  │
│  │                                                                   │  │
│  │  Stage 0-3:  读取 ACC slice (4 cy, 256 B/cy)                     │  │
│  │  Stage 4:    缓冲                                                 │  │
│  │  Stage 5-6:  nz → nd 格式转换 (2 cy)                             │  │
│  │  Stage 7:    量化 (FP32→FP16/FP8/INT8) (1 cy)                    │  │
│  │  Stage 8:    rowmax (可选) (1 cy)                                │  │
│  │  Stage 9:    输出缓冲                                             │  │
│  │  Stage 10:   TileStore 请求                                      │  │
│  └────────────────────────┬─────────────────────────────────────────┘  │
│                           ↓                                             │
│  ┌──────────────────────────────────────────────────────────────────┐  │
│  │               TileStore (写回 TileReg)                            │  │
│  │                                                                   │  │
│  │  生成写请求 → TMU 仲裁 → TileReg 写入                             │  │
│  │  写完成 → 释放 ACC slice                                          │  │
│  └────────────────────────┬─────────────────────────────────────────┘  │
│                           ↓                                             │
│                      acccvtuop 完成                                      │
│                      所有 slices 完成 → acccvt 完成                      │
└─────────────────────────────────────────────────────────────────────────┘
```

## 3. 带宽和延迟总结

### 3.1 关键带宽

| 接口 | 带宽 | 说明 |
|------|------|------|
| TReg Burst (读) | 2048 B/cy | TMU → CUBE |
| L0A → MAC | 512 B/cy | A 操作数 |
| L0B → MAC | 512 B/cy | B 操作数（FP4: 1024 B/cy） |
| MAC → BufferC | 1 KB/cy | 16×16×FP32 |
| BufferC → ACC | 256 B/cy | RMW 写 |
| ACC → FixPipe | 256 B/cy | 读取 |
| TileStore | 2048 B/cy | CUBE → TMU |

### 3.2 关键延迟

| 操作 | 延迟 | 说明 |
|------|------|------|
| TReg 读取 | ~5 cy | 预取 |
| L0 Cache Hit | 1-2 cy | 查找 + 读取 |
| L0 Cache Miss | 8-12 cy | TileLoad |
| MAC 计算 | ~7 cy | 脉动阵列 |
| BufferC 累加 | 1 cy | FP32 加法 |
| ACC RMW | 9 cy | 读(4) + 加(1) + 写(4) |
| FixPipe | ~11 cy | Per slice |
| TileStore | ~3 cy | 请求 + 写入 |

### 3.3 单个 uop 延迟

```
无 ACC RMW（K_idx 非 chunk 结束）：
  TReg(5) + L0(2) + MAC(7) + BufC(1) = ~15 cycles

有 ACC RMW（K_idx 是 chunk 结束）：
  TReg(5) + L0(2) + MAC(7) + BufC(1) + ACC_RMW(9) = ~24 cycles
```

## 4. 性能特征

### 4.1 峰值性能（@ 1.5 GHz）

| 格式 | MACs | TFLOPS/TOPS |
|------|------|-------------|
| FP16 | 4096 | 12.3 TFLOPS |
| FP8  | 8192 | 24.6 TOPS |
| FP4  | 32768 | 98.3 TOPS |

### 4.2 实际性能

**MAC 阵列利用率**：
- K=1 (小矩阵)：~40%
- K=4：~70%
- K≥16 (大矩阵)：~85-90%

**瓶颈**：
- 小矩阵：TileReg 读取延迟
- 大矩阵：接近峰值，受限于 L0 Cache 命中率

### 4.3 L0 Cache 命中率

| 工作负载 | L0A 命中率 | L0B 命中率 |
|----------|-----------|-----------|
| 16×16 | ~98% | ~98% |
| 64×64 | ~90% | ~90% |
| 128×128 | ~75% | ~75% |
