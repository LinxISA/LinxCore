## 2. CUBE 流水线阶段详图

```
阶段      功能                                     延迟      数据流
════════════════════════════════════════════════════════════════════════════

           ┌─────────────────────────────────────┐
Stage 0    │  命令解码 (CMD)                     │   1      命令载荷
           │  - 解析 OPA/DRAIN                   │          ↓
           │  - 提取操作数 (zd, Ta, Tb, K)      │
           └─────────────────┬───────────────────┘
                             ↓
           ┌─────────────────────────────────────┐
Stage 1    │  Tile 请求 (TREQ)                   │   1      tile 索引
           │  - 发起 TMU 读请求                  │          ↓
           │  - Ta, Tb → TMU                     │
           └─────────────────┬───────────────────┘
                             ↓
           ┌─────────────────────────────────────┐
Stage 2-4  │  Tile 读取 (TREAD)                  │  3-5     TReg burst
           │  - TMU 仲裁                         │          ↓
           │  - TRegFile 访问                    │        2048 B/cy
           │  - Burst 传输                       │
           └─────────────────┬───────────────────┘
                             ↓
           ┌─────────────────────────────────────┐
Stage 5    │  Burst Buffer (BBUF)                │   1      2048 B
           │  - 接收 burst                       │          ↓
           │  - Ping-pong 切换                   │        8 KB
           └─────────────────┬───────────────────┘
                             ↓
           ┌─────────────────────────────────────┐
Stage 6    │  L0A/L0B Staging (L0)               │   1      256 B/cy
           │  - 缓存 A/B 操作数                  │          ↓
           │  - 数据复用                         │        1KB + 1KB
           │  - 分发到 8 banks                   │
           └─────────────────┬───────────────────┘
                             ↓
           ┌─────────────────────────────────────┐
Stage 7    │  MAC 输入寄存 (MREG)                │   1      A[8], B[64]
           │  - 每个 bank: A[8] × B[64]          │          ↓
           └─────────────────┬───────────────────┘
                             ↓
           ┌─────────────────────────────────────┐
Stage 8    │  乘法器 (MUL)                       │   1      8×64×8 = 4096
           │  - 4096 个并行乘法器                │          ↓
           │  - A × B                            │        部分积
           └─────────────────┬───────────────────┘
                             ↓
           ┌─────────────────────────────────────┐
Stage 9    │  加法器 (ADD)                       │   1      4096 个和
           │  - 4096 个并行加法器                │          ↓
           │  - partial_sum += A × B             │
           └─────────────────┬───────────────────┘
                             ↓
           ┌─────────────────────────────────────┐
Stage 10   │  MAC 输出寄存 (MACC)                │   1      8×64×8
           │  - 寄存 MAC 结果                    │          ↓
           └─────────────────┬───────────────────┘
                             ↓
           ┌─────────────────────────────────────┐
Stage 11   │  归约树 L1 (RED1)                   │   1      Intra-bank
           │  - Bank 内归约 (8→4)                │          ↓
           └─────────────────┬───────────────────┘
                             ↓
           ┌─────────────────────────────────────┐
Stage 12   │  归约树 L2 (RED2)                   │   1      Intra-bank
           │  - Bank 内归约 (4→2)                │          ↓
           └─────────────────┬───────────────────┘
                             ↓
           ┌─────────────────────────────────────┐
Stage 13   │  归约树 L3 (RED3)                   │   1      Intra-bank
           │  - Bank 内归约 (2→1)                │          ↓
           │  - 每个 bank 产生 64 列和           │        8×64 列和
           └─────────────────┬───────────────────┘
                             ↓
           ┌─────────────────────────────────────┐
Stage 14   │  Inter-bank 归约 (REDB)             │  0-2     (可选)
           │  - 跨 bank 归约                     │          ↓
           │  - 仅大输出形状需要                 │
           └─────────────────┬───────────────────┘
                             ↓
           ┌─────────────────────────────────────┐
Stage 15   │  KACC 累加 (KACC)                   │   1      8×64 × FP32
           │  - 中间累加                         │          ↓
           │  - K-chunk 管理                     │        2 KB
           └─────────────────┬───────────────────┘
                             ↓
           ┌─────────────────────────────────────┐
Stage 16   │  pAcc 读取 (PREAD)                  │   1      RMW 读阶段
           │  - 读取当前累加器                   │          ↓
           └─────────────────┬───────────────────┘
                             ↓
           ┌─────────────────────────────────────┐
Stage 17   │  pAcc 累加 (PACC)                   │   1      FP32 加法
           │  - pAcc += KACC                     │          ↓
           └─────────────────┬───────────────────┘
                             ↓
           ┌─────────────────────────────────────┐
Stage 18   │  pAcc 写回 (PWRITE)                 │   1      RMW 写阶段
           │  - 写回累加器                       │          ↓
           └─────────────────┴───────────────────┘

════════════════════════════════════════════════════════════════════════════
总延迟：~18-20 cycles/K-iteration（CUBE.OPA）
```

## 4. CUBE.DRAIN 流水线

```
Cycle    阶段                   说明
────────────────────────────────────────────────────────────────
0        CMD                    解码 DRAIN 命令
1        RAT 查询               查询累加器映射
2        PREAD                  读取 pAcc
3        PREAD                  (并行读取所有 banks)
4        ZZ→ND 转换             FixPipe 格式转换
5        ZZ→ND 转换             重排序
6        格式转换               FP32 → FP16/BF16/...
7        紧凑打包               合并半 tiles
8        Ping-pong Buffer       写入输出缓冲
9        Ping-pong Buffer       填充缓冲
10       TReg 请求              发起 TMU 写请求
11       TMU 仲裁               等待 TMU
12       TReg Burst             传输第 1 个 2048 B
13       TReg Burst             传输第 2 个 2048 B
14       TRegFile 写入          写入 TRegFile
15       写完成                 TMU 确认
16       RAT 释放               释放累加器
17       响应                   向 BROB 报告完成
────────────────────────────────────────────────────────────────
总延迟：~17 cycles（CUBE.DRAIN）
```

## 6. 关键路径分析

```
临界路径 1: MAC + 归约 + KACC
───────────────────────────────────────────────────────────────────
MREG → MUL(1cy) → ADD(1cy) → MACC(1cy) → RED1(1cy) → RED2(1cy) 
→ RED3(1cy) → KACC(1cy)

总延迟: 7 cycles
频率限制: ~1.5 GHz (假设每级 ~0.66 ns)


临界路径 2: pAcc RMW
───────────────────────────────────────────────────────────────────
PREAD(1cy) → PACC(1cy) → PWRITE(1cy)

总延迟: 3 cycles
频率限制: ~1.8 GHz


临界路径 3: FixPipe 格式转换
───────────────────────────────────────────────────────────────────
PREAD(1cy) → ZZ→ND(2cy) → FP32→FP16(1cy) → Pack(1cy)

总延迟: 5 cycles
频率限制: ~1.6 GHz
```

## 7. 并发和流水线利用率

### 7.1 K 循环流水线

```
理想情况（无停顿）：
───────────────────────────────────────────────────────────────────
K=0: [0  ][1-4 ][5  ][6-15][16-18]
K=1:      [0  ][1-4 ][5  ][6-15][16-18]
K=2:           [0  ][1-4 ][5  ][6-15][16-18]
K=3:                [0  ][1-4 ][5  ][6-15][16-18]

启动间隔: 5 cycles（受 TReg 读取限制）


实际情况（有停顿）：
───────────────────────────────────────────────────────────────────
K=0: [0-5  ][6-15][16-18]
K=1:        [0-5  ][6-15][16-18]
K=2:               [0-5  ][6-15][16-18]
K=3:                      [0-5  ][6-15][16-18]

启动间隔: ~10 cycles（考虑 TMU 仲裁和竞争）
```
